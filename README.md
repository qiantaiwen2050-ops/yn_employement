# 云南省企业就业失业数据采集系统

软件项目管理实现项目。基于 SRS《软件需求规格说明书》，按"核心闭环 + 关键亮点"范围实现，前后端分离、本地可跑、不依赖外网。

## 技术栈

| 层 | 选型 |
|---|---|
| 后端 | Spring Boot 3.2 · MyBatis-Plus · SQLite · JWT (jjwt) · Hutool BCrypt |
| 前端 | Vue 3 · Vite · Element Plus · Pinia · Vue Router · axios · ECharts |
| 数据库 | SQLite (`data/yn-employment.db`，启动时自动建表 + 灌入字典) |
| JDK / Node | JDK 17 (Temurin) · Node 20 |

> 第一次运行会自动：建库、建表、灌入字典数据、创建 3 个演示账号。

## 目录结构

```
yn-employment/
├── backend/      Spring Boot 后端
├── frontend/     Vite + Vue 3 前端
├── data/         SQLite 数据库文件存放目录（首次启动后生成）
├── env.sh        本机 JDK / Maven / Node 路径快捷设置（仅本机适用）
└── README.md
```

## 环境准备（已在本机配置完毕）

```bash
# JDK 17 / Maven 3.9 / Node 20 均已下载到 ~/devtools/，本项目根有 env.sh：
source env.sh   # 在新终端里需要先 source 一次
```

如换机器/换路径，编辑 `env.sh` 调整即可，或自行装好上述版本。

## 启动

**两个终端分别启动（每个终端先 `source env.sh`）：**

终端 A — 后端：
```bash
cd backend
mvn spring-boot:run
# 启动成功后监听 http://localhost:8080/api
```

终端 B — 前端：
```bash
cd frontend
npm run dev
# Vite dev server 监听 http://localhost:5173
# 已配置代理：前端 /api/* → 后端 :8080
```

打开 http://localhost:5173 即可。

## 配置管理（CM）

完整 CM 计划见 [docs/CM-Plan.md](docs/CM-Plan.md)，所有版本变更见 [CHANGELOG.md](CHANGELOG.md)。

**简要说明**：
- **Git + GitHub** 做版本控制，远程仓库 https://github.com/qiantaiwen2050-ops/yn_employement
- **分支策略**：`main` 受保护，所有变更通过 `<type>/<topic>` 分支 + Pull Request 合入
- **里程碑基线**：M1–M7 每完成一个打基线 tag（`v0.1.0-M1` ... `v0.7.0-M7`），首次正式发布为 `v1.0.0`
- **持续集成**：[.github/workflows/ci.yml](.github/workflows/ci.yml) 在每次 push / PR 自动跑后端 Maven compile + 前端 Vite build
- **应用层审计**：`sys_log` 表记录所有"有后果"的写操作，省级用户可在「系统管理 → 操作日志」页查询
- **变更控制流程**：`Issue → 分支 → commits → PR → CI 通过 → Code Review → Merge → 关闭 Issue + 删分支`

## 已知约束（与 SRS / SOW 的差异）

- SRS 第 7 章约定 MySQL 8.0+，本项目用 SQLite 简化部署，迁移成本极低（改 `application.yml` 数据源即可）
- SRS 提到的 Docker / Nginx / Redis / ELK / Prometheus 不在本作业范围内
- 国家失业监测接口（4.2 节）按"本地 Mock 服务"实现，可演示完整上报-反馈-重试链路；mock 内置 ~10% 概率失败用于演示重试流程
- **国家接口自动定时上报**未启用（M6「国家接口 → 自动上报设置」是占位 UI；@Scheduled job 未注册）— 真要上线只需注册一个 `@Scheduled` 方法定时调用 `NationReportService.uploadPeriod`
- **通知未接富文本编辑器**（SRS §4.7.3 建议但未强制）— 当前用 textarea，保留换行
- **就业失业原因父级映射**（SOW 表"就业失业原因"列出的 super_reason 关系）未实现 — 影响真正接国家系统时的代码翻译，mock 场景下不暴露
